## 2. DataLoader Optimization Tactics

### Core DataLoader Parameters

```python
from torch.utils.data import DataLoader

dataloader = DataLoader(
    dataset,
    batch_size=32,
    num_workers=4,           # CPU parallelization
    pin_memory=True,         # Faster CPU->GPU transfer
    prefetch_factor=2,       # Batches to prefetch per worker
    persistent_workers=True, # Keep workers alive between epochs
    multiprocessing_context='fork'  # or 'spawn' on Windows
)
```

**Key tactics:**

1. **`num_workers`**: Start with `4-8`, or `min(cpu_count, 8)`. Too many workers cause overhead.
   - Monitor CPU usage to find the sweet spot
   - If CPU-bound: increase workers
   - If GPU-bound: workers won't help much

2. **`pin_memory=True`**: Allocates data in pinned (page-locked) memory for faster CPU→GPU transfer via DMA
   - Use when GPU is bottleneck
   - Increases CPU memory usage

3. **`prefetch_factor`**: Number of batches each worker prefetches (default=2)
   - Increase to 4-8 if GPU is waiting for data
   - Decrease if RAM is limited

4. **`persistent_workers=True`**: Keeps worker processes alive between epochs
   - Eliminates worker initialization overhead
   - Essential for fast epochs

### Bucketing/Dynamic Batching

For variable-length sequences (NLP, time series):

```python
from torch.utils.data import Sampler
from torch.nn.utils.rnn import pad_sequence

class BucketBatchSampler(Sampler):
    """Group similar-length samples to minimize padding"""
    def __init__(self, data_source, batch_size, drop_last=False):
        self.data_source = data_source
        self.batch_size = batch_size

        # Sort by sequence length
        self.indices = sorted(range(len(data_source)),
                            key=lambda i: len(data_source[i]))

    def __iter__(self):
        for i in range(0, len(self.indices), self.batch_size):
            yield self.indices[i:i + self.batch_size]

    def __len__(self):
        return (len(self.indices) + self.batch_size - 1) // self.batch_size

# Use with DataLoader
sampler = BucketBatchSampler(dataset, batch_size=32)
dataloader = DataLoader(dataset, batch_sampler=sampler, num_workers=4)
```

### GPU Streams for Concurrent Execution

```python
import torch

# Create separate streams for data transfer and computation
data_stream = torch.cuda.Stream()
compute_stream = torch.cuda.Stream()

for batch_idx, (data, target) in enumerate(dataloader):
    # Overlap data transfer with previous batch computation
    with torch.cuda.stream(data_stream):
        data = data.cuda(non_blocking=True)
        target = target.cuda(non_blocking=True)

    # Wait for data to be ready
    torch.cuda.current_stream().wait_stream(data_stream)

    # Compute
    output = model(data)
    loss = criterion(output, target)
```

## 3. Beyond DataLoader: CPU/GPU Utilization Strategies

### A. Mixed Precision Training (AMP)
Reduces memory usage and increases throughput:

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()

    # Forward pass in fp16
    with autocast():
        output = model(data)
        loss = criterion(output, target)

    # Backward pass with scaled gradients
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

**Benefits:**
- 2-3x faster on Tensor Cores (A100, V100, RTX GPUs)
- Reduces memory by ~50%, allowing larger batches

### B. Gradient Accumulation
Simulate larger batches when GPU memory is limited:

```python
accumulation_steps = 4

optimizer.zero_grad()
for i, (data, target) in enumerate(dataloader):
    output = model(data)
    loss = criterion(output, target) / accumulation_steps
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

### C. Asynchronous Data Preprocessing

Offload heavy preprocessing to CPU while GPU trains:

```python
import torch.multiprocessing as mp
from queue import Queue

class AsyncPreprocessor:
    def __init__(self, dataloader, transform_fn, queue_size=5):
        self.queue = mp.Queue(maxsize=queue_size)
        self.process = mp.Process(target=self._worker,
                                  args=(dataloader, transform_fn))
        self.process.start()

    def _worker(self, dataloader, transform_fn):
        for batch in dataloader:
            processed = transform_fn(batch)
            self.queue.put(processed)
        self.queue.put(None)  # Signal completion

    def __iter__(self):
        while True:
            batch = self.queue.get()
            if batch is None:
                break
            yield batch
```

### E. Optimize Data Pipeline

```python
class EfficientDataset(torch.utils.data.Dataset):
    def __init__(self, data_path):
        # Load all data into memory if possible
        self.data = self._load_all_data(data_path)

        # Pre-compute expensive transformations
        self.preprocessed = self._preprocess_all()

    def __getitem__(self, idx):
        # Only cheap operations here
        return self.preprocessed[idx]
```

**Tactics:**
- Cache preprocessed data to disk (HDF5, LMDB)
- Use memory mapping for large datasets (`np.memmap`)
- Avoid expensive operations in `__getitem__` (I/O, decoding)

## Key Diagnostic Questions

**If GPU utilization is low (<80%):**
- Are you CPU-bound? → Increase `num_workers`, optimize Dataset `__getitem__`
- Small batches? → Increase batch size or use gradient accumulation
- Data transfer bottleneck? → Enable `pin_memory=True`, use `non_blocking=True`
- Model too small? → Use mixed precision, compile model

**If CPU utilization is low:**
- Too few workers → Increase `num_workers`
- Lightweight preprocessing → May not need many workers
- GPU is the bottleneck → This is fine!

**If both are underutilized:**
- Disk I/O bottleneck → Cache data in RAM, use faster storage (SSD/NVMe)
- Inefficient data loading → Profile `__getitem__`, optimize preprocessing
- Synchronization issues → Use asynchronous data loading, CUDA streams

The goal is typically **high GPU utilization** (>90%) since that's usually your most expensive resource!
