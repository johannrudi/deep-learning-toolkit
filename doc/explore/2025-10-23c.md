Below is a concise, architecture-specific study of how to inject Gaussian noise tensors
into generators, with practical patterns, shapes, and pitfalls. I include minimal
reference implementations and recommendations tailored to MLPs, CNNs/UNets, and
Transformers, plus cross-cutting best practices.

## Core idea by architecture

- MLPs: noise is a global vector z∈R^d. Feed as a feature vector at input; optionally
  re-inject at intermediate layers or modulate layers via FiLM-style parameters.
- CNNs/UNets: noise should have spatial structure for rich textures. Use a global z to
  create an initial feature map (via FC+reshape/broadcast), and optionally add per-layer
  per-pixel noise maps with learnable scales. Condition via channel-wise concatenation
  or SPADE/FiLM.
- Transformers: noise should be tokens. Expand z into a sequence of latent tokens (with
  positional embeddings), or use z-derived style tokens for cross-attention/modulation.
  In diffusion, the “noise” is in x_t; the net consumes (noisy sample, time embedding)
  rather than a standalone z.

### SPADE and FiLM

**SPADE** (Spatially-Adaptive Normalization) and **FiLM** (Feature-wise Linear
Modulation) are conditioning techniques for neural networks:

**SPADE**: Modulates normalization parameters (mean/variance shifts and scales)
spatially using a learned convolution from conditioning information. Each spatial
location gets different normalization parameters based on the condition (e.g., semantic
maps). Introduced in "Semantic Image Synthesis with Spatially-Adaptive Normalization"
(Park et al., 2019).

**FiLM**: Applies affine transformations per feature channel using learned scale (γ) and
shift (β) parameters from conditioning input: `FiLM(F) = γ ⊙ F + β`. The modulation is
channel-wise but spatially uniform. Introduced in "FiLM: Visual Reasoning with a General
Conditioning Layer" (Perez et al., 2018).

Both inject conditioning information into intermediate network layers. SPADE is
spatially-varying (different modulation per pixel), while FiLM is spatially-uniform
(same modulation across all spatial positions in a channel).

## MLPs (vector in, vector out or small tensor)

- Primary patterns
  - Direct input: g(y, z) = MLP([y, z]) for conditional GANs; just g(z) for
    unconditional.
  - Multi-layer injection: pass small noise vectors to specific hidden layers (improves
    stochasticity while keeping early layers semantically stable).
  - Modulation: map z→w and produce scale/shift for layer norms or linear layers (FiLM).
    This is a style-based alternative to concatenation.
- Shapes
  - z: [B, d_z]; concatenate with any conditioning y on feature dim. Flatten other small
    tensors before concat.
- Tips
  - Use moderate d_z (64–512). Normalize inputs (zero mean, unit variance). If outputs
    are bounded, use output activations (e.g., Tanh).
  - In your toolkit, MLPNet_MultIn and MLPResNet already support multi-input and hidden
    inputs via h0, h1, …; and generator signatures in dlk/opt/train_gan.py expect
    g_net(y, z).

Example: MLP generator taking condition y and noise z (concat at input; optional hidden
injection)
```python
class MLPGenerator(nn.Module):
    r"""MLP generator: x = g(y, z)."""
    def __init__(self, y_dim, z_dim, x_dim, hidden=[256,256], act=nn.GELU()):
        super().__init__()
        layers = []
        in_dim = y_dim + z_dim
        for h in hidden:
            layers += [nn.Linear(in_dim, h), act]
            in_dim = h
        self.backbone = nn.Sequential(*layers)
        self.head = nn.Linear(in_dim, x_dim)

    def forward(self, y, z):
        # concatenate inputs
        h = torch.cat([y.flatten(1), z], dim=1)
        h = self.backbone(h)
        x = self.head(h)
        return x
```

## CNNs/UNets (images, signals; spatial outputs)

- Primary patterns
  - Global z → initial feature map: map z with a linear layer to C0×H0×W0 (or 1D length
    L0), reshape, then upsample via conv blocks to target size.
  - Per-layer noise injection: add Gaussian noise maps ε_l∼N(0,1) of shape [B,1,H_l,W_l]
    to feature maps at each block, scaled by learnable α_l (as in StyleGAN, adds fine
    stochastic details).
  - Channel concat: broadcast z over spatial dims (tile to [B,Cz,H,W]) and concat on
    channels at chosen layers. Good for strong conditioning but can be heavy.
  - Conditional modulation: SPADE/FiLM to modulate normalization with y (preferred vs
    raw concat for structured conditions).
- Shapes
  - z: [B, d_z] mapped to [B, C0, H0, W0] (2D) or [B, C0, L0] (1D). Typical
    C0∈[256,512], H0=W0∈{4,8}, or L0∈{16,32}.
- Tips
  - Prefer interpolate+conv over transposed conv to avoid checkerboard artifacts.
  - Zero-init final conv’s weights/bias for stable starts; use small noise scales
    initially.
  - For UNets, per-layer noise maps can be added at decoder blocks; conditioning can
    flow through skip connections.

Example: 2D conv generator with global z and per-layer noise injection
```python
class Conv2DGenerator(nn.Module):
    r"""Image generator: x = g(y, z); global z -> seed map, per-layer noise maps with learned scales."""
    def __init__(self, z_dim=256, base_ch=256, img_ch=3, size=64):
        super().__init__()
        self.H0 = self.W0 = 4
        self.fc = nn.Linear(z_dim, base_ch * self.H0 * self.W0)
        self.blocks = nn.ModuleList([
            nn.Sequential(nn.Upsample(scale_factor=2, mode='nearest'),
                          nn.Conv2d(base_ch, base_ch//2, 3, padding=1), nn.GELU()),
            nn.Sequential(nn.Upsample(scale_factor=2, mode='nearest'),
                          nn.Conv2d(base_ch//2, base_ch//4, 3, padding=1), nn.GELU()),
            nn.Sequential(nn.Upsample(scale_factor=2, mode='nearest'),
                          nn.Conv2d(base_ch//4, base_ch//8, 3, padding=1), nn.GELU()),
        ])
        self.noise_scales = nn.ParameterList([nn.Parameter(torch.zeros(1)) for _ in self.blocks])
        self.to_rgb = nn.Conv2d(base_ch//8, img_ch, 1)

    def forward(self, y, z):
        # map z to seed map
        h = self.fc(z).view(z.size(0), -1, self.H0, self.W0)
        # upsample with per-layer noise injection
        for i, blk in enumerate(self.blocks):
            h = blk(h)
            eps = torch.randn(h.size(0), 1, h.size(2), h.size(3), device=h.device, dtype=h.dtype)
            h = h + self.noise_scales[i] * eps  # add spatial noise
        x = torch.tanh(self.to_rgb(h))
        return x
```

## Transformers (token-based generation; images, sequences)

- Primary patterns
  - Latent-token expansion: map z→T latent tokens (T equals number of output tokens, e.g., image patches). Add positional embeddings, process with transformer blocks, then decode tokens to pixels/samples.
  - Style/cross-attention: derive a “style token” (or several) from z; use cross-attention where content tokens attend to style tokens; or modulate MSA/MLP via FiLM/gating from z (“StyleFormer”-like).
  - Diffusion case: the net consumes x_t (noisy data) and t; you don’t pass an explicit z—the Gaussian noise corrupts x, and you condition on t.
- Shapes
  - z: [B, d_z]; expand to tokens [B, T, d_model] or to K style tokens [B, K, d_model]. Ensure positional embeddings for content tokens; style tokens typically have no position or use dedicated embeddings.
- Tips
  - Pre-LN transformers with residual scaling improve stability; initialize final projections near zero.
  - Keep T manageable (patching reduces T). For 2D, T = (H/P)×(W/P).
  - For conditional tasks, feed y as dedicated tokens or via cross-attention; avoid naive concat of y to every token without modulation.

Example: transformer generator using z-expanded latent tokens
```python
class TransformerGenerator(nn.Module):
    r"""Token generator: expand z to T latent tokens, transform, decode to image patches."""
    def __init__(self, z_dim=256, d_model=512, T=256, n_blocks=8, n_heads=8, patch_size=4, img_ch=3):
        super().__init__()
        self.T = T
        self.patch_size = patch_size
        self.latent_proj = nn.Linear(z_dim, T * d_model)
        self.pos_embed = nn.Parameter(torch.randn(1, T, d_model) * 0.02)
        self.blocks = nn.ModuleList([
            nn.Sequential(nn.LayerNorm(d_model),
                          nn.MultiheadAttention(d_model, n_heads, batch_first=True),
                          nn.LayerNorm(d_model),
                          nn.Sequential(nn.Linear(d_model, 4*d_model), nn.GELU(), nn.Linear(4*d_model, d_model)))
            for _ in range(n_blocks)
        ])
        self.norm = nn.LayerNorm(d_model)
        self.to_patches = nn.Linear(d_model, img_ch * patch_size * patch_size)

    def forward(self, y, z):
        # create latent token sequence from z
        x = self.latent_proj(z).view(z.size(0), self.T, -1) + self.pos_embed
        # transformer blocks with residuals
        for blk in self.blocks:
            # apply attention
            ln1, attn, ln2, mlp = blk
            h = x + attn(ln1(x), ln1(x), ln1(x), need_weights=False)[0]
            x = h + mlp(ln2(h))
        x = self.norm(x)
        # decode tokens to patches and fold to image
        patches = self.to_patches(x)  # [B, T, C*P*P]
        # fold: caller should reshape to (B, C, H, W) with correct grid for T and patch_size
        return patches
```

## How to choose and scale noise

- Distribution
  - Standard z∼N(0,I) is default. Truncated normal improves sample quality at cost of
    diversity; heavy-tailed priors may encourage diversity but can destabilize training.
- Global vs local noise
  - Global z controls coarse structure; per-layer spatial noise (CNN/UNet) adds
    microtexture. Early-layer noise affects global layout; late-layer noise affects fine
    detail—inject accordingly.
- Conditioning
  - Prefer modulation (FiLM/SPADE) over raw concatenation for structured conditions
    (e.g., segmentation masks), especially in CNNs/UNets. In transformers, use
    cross-attention or condition tokens.
- Normalization and scale
  - Keep noise magnitude balanced: small learnable scales for injected noise; normalize
    features (LayerNorm/GroupNorm); avoid overwhelming conditional inputs.
- Training stability
  - Zero-init last layer in generators, spectral norm or gradient penalty in
    discriminators (your dlk has Wasserstein/GP utilities). Prefer upsample+conv over
    transposed conv.

## Practical recipes by task

- Tabular/time-series vector output (small): MLP with input concat of [y, z]; optionally
  inject extra small noise at one late hidden layer.
- 1D signal generation: Conv1D generator with z→[B,C0,L0], upsample to target length;
  add per-layer 1D noise [B,1,L].
- Images: Conv2D/UNet generator with z→[B,C0,H0,W0], upsample stack; add per-layer 2D
  noise; condition via SPADE/FiLM if you have structured y.
- Large images/sequences where global dependencies matter: Transformer with z-expanded
  latent tokens and cross-attend to condition tokens; decode to patches and fold.

## Diagnostics and pitfalls

- If samples lack fine details: add or strengthen per-layer spatial noise; inject later
  in the network.
- If global structure is unstable: reduce early noise injection; increase d_z and route
  z earlier; add mapping network z→w.
- If conditioning is ignored: switch from concat to modulation (CNN: SPADE/FiLM;
  Transformer: cross-attention); increase conditioner capacity.
- Checkerboard artifacts: replace transposed conv with interpolate+conv.
- Training collapse: balance D/G steps, add regularizers (R1/GP), tune noise scale.
