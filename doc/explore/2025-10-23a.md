Below is a compact, practical “single-device” playbook: what actually speeds training,
which PyTorch packages implement it, and how to build fast, correct custom dataloaders.

## Fast on 1 GPU/1 CPU: levers that matter most

- Reduce per-step cost: PyTorch 2.x compile (torch.compile), mixed precision (AMP,
  TF32), fused kernels (FlashAttention/xFormers, fused optimizers), channels_last for
  CNNs, efficient attention, kernel fusion.
- Improve utilization: dataloader throughput (num_workers, persistent_workers,
  pin_memory, prefetch_factor, fast collate), GPU-side augmentations (Kornia, DALI),
  streaming shards (WebDataset, torchdata).
- Converge in fewer steps: good schedules (cosine/OneCycle warmup), strong baselines
  (SGD+momentum or AdamW), modern optimizers (Lion, RAdam, AdaBelief, LAMB/LARS),
  curvature approximations (K-FAC, Shampoo, Sophia, AdaHessian) when step cost is
  acceptable.
- Smart recipes: gradient accumulation to reach effective large batch; gradient
  clipping; EMA of weights; label smoothing + mixup/cutmix for stability; early
  stopping.

## PyTorch-native features to enable first

- torch.compile(model, mode='max-autotune') for stable shapes; cudnn.benchmark=True for
  CNNs; torch.set_float32_matmul_precision('high'); allow TF32 on Ampere+
  (torch.backends.cuda.matmul.allow_tf32=True).
- AMP: torch.cuda.amp.autocast + GradScaler; channels_last for convnets;
  torch.nn.functional.scaled_dot_product_attention uses fast kernels automatically if
  available.
- Checkpointing (torch.utils.checkpoint) to trade memory for speed via larger batches.

## Packages that implement fast training algorithms in PyTorch

- Optimizers (first-order, large-batch, curated):
  - pytorch-optimizer (jettify/pytorch-optimizer): AdamW, RAdam, Lookahead, Ranger,
    Lion, AdaBelief, NovoGrad, Yogi, LARS, LAMB, SGDP, DiffGrad, etc. One-stop shop to
    try modern variants.
  - bitsandbytes: 8-bit Adam/Lion/Adagrad; fused CUDA kernels; big memory bandwidth
    savings → higher throughput on a single GPU.
  - NVIDIA Apex: FusedAdam, FusedLAMB, AMP (alternate to native AMP); still useful for
    fused optimizers.
  - timm: battle-tested training recipes, create_optimizer_v2, schedulers (cosine,
    OneCycle), EMA, mixup/cutmix, fast data utilities.
- Curvature/second-order approximations (fewer steps, heavier per-step):
  - K-FAC: kfac_pytorch (academia-maintained repos exist); strong on medium-size models.
  - Shampoo: shampoo-pytorch (PyTorch implementations available).
  - AdaHessian: adahessian-pytorch.
  - Sophia: multiple PyTorch repos exist; try for large-scale models if step cost is
    acceptable.
- Attention and fused kernels:
  - flash-attn (FlashAttention v2): drop-in fast SDPA; large speedups for transformers.
  - xformers: memory-efficient SDPA and fused MLP/ops; works across many shapes.
- Data pipeline acceleration:
  - WebDataset: tar-sharded streaming I/O; avoids small-file bottlenecks; simple scaling
    of workers.
  - torchdata (DataPipes): composable streaming transforms; integrates with DataLoader.
  - NVIDIA DALI: GPU-accelerated decoding/augmentations (images/video); very effective
    for image pipelines.
  - Kornia: on-GPU differentiable augmentations; offload CPU bottlenecks for vision.

## How to handle data loading with custom Dataset objects (fast, correct)

- Choose the right dataset type:
  - Map-style Dataset for random access datasets that fit in indexed storage (images,
    arrays, LMDB, tar-shards).
  - IterableDataset for streams (webdataset, very large corpora) or when random access
    is impractical.
- Implement Dataset with low Python overhead:
  - Pre-open resources (index, memmap) in __init__; keep __getitem__ minimal.
  - Use memory-mapped arrays or tar/LMDB/zarr to avoid “lots of small files.”
  - Return contiguous tensors of fixed shapes; prefer tuples over dicts when speed-bound.
- Configure DataLoader for throughput:
  - num_workers: benchmark; for GPU training, often 4–8 per GPU for I/O-heavy pipelines;
    for CPU training, fewer (you share CPU with the model).
  - persistent_workers=True, prefetch_factor=2–4, pin_memory=True (GPU only),
    pin_memory_device='cuda', drop_last=True for stable shapes.
  - Use a fast collate_fn (stack already-typed tensors; avoid per-sample Python).
  - Use .to(device, non_blocking=True) with pinned memory; overlap host→device copies.
- Move heavy transforms off the CPU path:
  - For vision, decode minimally on CPU then augment on GPU (Kornia) or use DALI
    end-to-end.
  - If CPU transforms are required, batch them, use vectorized NumPy, or JIT-compiled
    libraries where possible.
- Prefer streaming/sharded formats for large data:
  - TAR+WebDataset or DataPipes sharding avoids filesystem overhead and simplifies
    multi-worker reads.
- Reproducibility and worker init:
  - Define worker_init_fn to seed NumPy/Python/torch; avoid global RNG in dataset state.

## Reference snippets

- Custom map-style dataset and DataLoader tuned for GPU training:

```python
import os, numpy as np, torch
from torch.utils.data import Dataset, DataLoader

class MemmapArrayDataset(Dataset):
    """
    A fast map-style dataset backed by NumPy memmap.
    Args:
        x_path: path to float32 features memmap of shape (N, C, H, W) or (N, D)
        y_path: path to labels memmap (optional)
        transform: optional callable applied after tensor conversion
    """
    def __init__(self, x_path, y_path=None, transform=None, dtype=torch.float32):
        self.x = np.memmap(x_path, mode='r')
        self.x = self.x.view(np.float32)  # ensure dtype
        self.shape = np.load(x_path + ".shape.npy")  # store once via preprocessing
        self.x = self.x.reshape(*self.shape)
        self.y = None
        if y_path is not None:
            self.y = np.memmap(y_path, mode='r').view(np.int64)
        self.transform = transform
        self.dtype = dtype

    def __len__(self): return self.shape[0]

    def __getitem__(self, i):
        # load a single sample with minimal Python overhead
        xi = torch.from_numpy(self.x[i]).to(self.dtype).contiguous()
        yi = None if self.y is None else int(self.y[i])
        if self.transform is not None:
            xi = self.transform(xi)
        return (xi, yi) if yi is not None else xi

def fast_collate(batch):
    # stack tensors and keep ints as tensors; avoid per-sample branching
    xs, ys = zip(*batch) if isinstance(batch[0], tuple) else (batch, None)
    x = torch.stack(xs, dim=0)
    if ys is None: return x
    y = torch.tensor(ys, dtype=torch.long)
    return x, y

def seed_worker(worker_id):
    # seed numpy/python/torch for reproducibility
    import random, numpy as np, torch
    seed = torch.initial_seed() % 2**32
    np.random.seed(seed); random.seed(seed)

loader = DataLoader(
    MemmapArrayDataset("data.x.mmap", "data.y.mmap"),
    batch_size=256,
    shuffle=True,
    num_workers=8,
    pin_memory=True,
    pin_memory_device='cuda',
    prefetch_factor=4,
    persistent_workers=True,
    collate_fn=fast_collate,
    worker_init_fn=seed_worker,
    drop_last=True,
)
```

- GPU-friendly training loop with compile + AMP + fused/8-bit optimizer fallback:

```python
import torch, torch.nn as nn

device = 'cuda'
model = nn.Sequential(nn.Linear(1024, 4096), nn.ReLU(), nn.Linear(4096, 10)).to(device)
model = torch.compile(model, mode='max-autotune')  # speed for stable shapes

torch.backends.cudnn.benchmark = True
torch.backends.cuda.matmul.allow_tf32 = True
torch.set_float32_matmul_precision('high')

try:
    import bitsandbytes as bnb
    optimizer = bnb.optim.AdamW8bit(model.parameters(), lr=3e-4, weight_decay=0.01)
except Exception:
    optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01)

scaler = torch.cuda.amp.GradScaler()
loss_fn = nn.CrossEntropyLoss()

for xb, yb in loader:
    # overlap H2D with non_blocking copies
    xb = xb.to(device, non_blocking=True)
    yb = yb.to(device, non_blocking=True)

    optimizer.zero_grad(set_to_none=True)
    with torch.cuda.amp.autocast(dtype=torch.bfloat16):
        logits = model(xb)
        loss = loss_fn(logits, yb)
    scaler.scale(loss).backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    scaler.step(optimizer)
    scaler.update()
```

- WebDataset-based pipeline (streaming TAR shards) for large image sets:

```python
import webdataset as wds, torch
from torch.utils.data import DataLoader

urls = "dataset/shards-{000000..000127}.tar"
dataset = (
    wds.WebDataset(urls, resampled=True)
      .shuffle(10000)
      .decode("pil")
      .to_tuple("jpg;png", "cls")
)

def collate(batch):
    xs, ys = zip(*batch)
    xs = torch.stack([torch.from_numpy(np.array(x)).permute(2,0,1).float()/255 for x in xs])
    ys = torch.tensor(ys, dtype=torch.long)
    return xs, ys

loader = DataLoader(dataset, batch_size=256, num_workers=8, pin_memory=True,
                    pin_memory_device='cuda', persistent_workers=True, collate_fn=collate)
```

## CPU-specific notes

- Use oneDNN (enabled by default); set torch.set_num_threads and
  torch.set_num_interop_threads to saturate cores.
- Avoid too many DataLoader workers (they compete with the model); pin_memory=False on
  CPU; consider bf16 if your CPU supports it (AVX512-BF16/AMX-BF16).
