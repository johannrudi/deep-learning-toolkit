You’re right to call out the ambiguity: **Hellinger distance is defined on probability distributions**, so you have to decide what “distribution” a histogram represents. If you feed raw counts in, *scaling absolutely changes the value* unless you deliberately remove that degree of freedom.

Here are practical, defensible ways to make Hellinger comparisons **invariant to arbitrary scaling** (and what each choice means).

---

## 1) Treat each histogram as a probability mass function (L1 normalize)

If “shape” matters and total mass doesn’t, the most standard move is:

[
p_i=\frac{x_i}{\sum_j x_j}, \qquad q_i=\frac{y_i}{\sum_j y_j}
]

Then Hellinger distance:

[
H(p,q)=\frac{1}{\sqrt{2}}\sqrt{\sum_i \left(\sqrt{p_i}-\sqrt{q_i}\right)^2}
]

**Why it fixes scaling:** multiplying all (x_i) by any constant cancels in the normalization, so the distance depends only on relative bin proportions.

**Gotchas**

* Needs **nonnegative** bins.
* If (\sum x_i = 0) (all bins zero), you need a convention (e.g., define as “empty”, skip, or add smoothing).

**When to use:** counts, frequencies, or any histogram that you want to compare by *composition*.

---

## 2) If bin widths differ: normalize to a density before L1 normalization

If histogram bins have different widths (w_i), raw counts aren’t directly comparable as a density. Convert to density first:

[
\tilde{x}_i = \frac{x_i}{w_i},\quad \tilde{y}_i=\frac{y_i}{w_i}
]
then L1 normalize (\tilde{x}, \tilde{y}) to sum to 1 and compute Hellinger.

**When to use:** variable bin widths, or when the histogram approximates a continuous PDF.

---

## 3) Choose the scaling that *minimizes* Hellinger distance (explicit scale-invariance)

If you truly want to ignore relative scale without committing to “probability distribution” semantics, you can define a **scale-invariant distance** by optimizing over a positive scalar (c):

[
d(x,y)=\min_{c>0} H!\left(\frac{x}{|x|_1}, \frac{c,y}{|c,y|_1}\right)
]

But note: if you L1-normalize inside, the (c) disappears, so this reduces to option (1).

If instead you want to compare **raw nonnegative vectors** using Hellinger-like geometry *without* forcing sum-to-1, you can optimize:

[
d_{\text{SI}}(x,y)=\min_{c>0}\left|\sqrt{x}-\sqrt{c,y}\right|_2
]

This has a closed-form minimizer:

[
c^*=\left(\frac{\sum_i \sqrt{x_i}\sqrt{y_i}}{\sum_i y_i}\right)^2
\quad (\text{for } \sum_i y_i>0)
]

Then use ( \left|\sqrt{x}-\sqrt{c^* y}\right|_2 ) (optionally scaled by (1/\sqrt{2})).

**Interpretation:** “How close can these be after allowing a single global gain on (y)?”
**When to use:** you believe one histogram may be a scaled version of the other (e.g., different exposure time, different sample size) and you want best alignment under that model.

---

## 4) Normalize with L2 instead of L1 if you want “direction-only” comparison in sqrt-space

Because Hellinger uses square-roots, another clean approach is:

1. map (u=\sqrt{x}), (v=\sqrt{y}) (elementwise)
2. L2-normalize: (\hat u = u/|u|_2), (\hat v=v/|v|_2)
3. compare via Euclidean distance (or cosine distance) in that space:

[
d(u,v)=|\hat u-\hat v|_2
]

This is equivalent to comparing the **angles** between (\sqrt{x}) and (\sqrt{y}), i.e., it is *fully invariant* to global scaling of (x) or (y).

**Pros:** very scale-robust, often numerically stable.
**Cons:** no longer exactly the textbook Hellinger distance between probability distributions; it’s “Hellinger-style geometry.”

---

## 5) Smoothing / pseudocounts to avoid zero-bin instability

If you normalize to probabilities, zeros are fine mathematically, but **empty histograms** or tiny totals can be problematic. Common fix:

[
x'_i = x_i + \epsilon,\quad y'_i=y_i+\epsilon
]
then normalize (usually L1) and compute Hellinger.

Use (\epsilon) as a small fraction of total counts (e.g., (10^{-12}) relative to sum, or (0.5) as a Jeffreys-style pseudocount if bins are counts).

---

## Which should you pick?

* **Most common / principled for histograms:** **(1) L1 normalize to PMFs** (and density-correct first if bin widths vary).
* If you want “same shape up to a single gain factor” on raw histograms: **(3) optimize the scale in sqrt-space**.
* If you want a simple, strongly scale-invariant similarity measure: **(4) L2-normalize in sqrt-space**.
